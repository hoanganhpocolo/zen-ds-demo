## Implementation Steps

1. **Add npm scripts to `packages/components/package.json`**
   ```json
   {
     "scripts": {
       "audit": "node scripts/audit-components.mjs",
       "audit:check": "node scripts/audit-components.mjs --check"
     }
   }
   ```

2. **Run full pipeline test**
   ```bash
   cd packages/components
   npm run audit
   # Verify audit-data.json generated with 40 components
   cat src/audit-data.json | node -e "const d=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8')); console.log(d.summary)"
   ```

3. **Test CI mode**
   ```bash
   npm run audit:check
   # Should exit 0 if no critical failures, or exit 1 with failure summary
   echo $?
   ```

4. **Test docs app renders**
   ```bash
   cd apps/docs
   npm run dev
   # Navigate to Audit page in sidebar, verify table renders
   ```

5. **Verify end-to-end flow**
   - Intentionally break a component (remove displayName) -> run `audit:check` -> expect exit 1
   - Fix it -> run `audit:check` -> expect exit 0
   - Run `audit` -> check JSON updated -> refresh docs -> dashboard reflects change

6. **Add `audit-data.json` to `.gitignore` consideration**
   - Decision: Do NOT gitignore. Committed JSON shows progress diffs in PRs.
   - Add a comment in the JSON noting it's auto-generated:
     The script should include `"_generated": "Auto-generated by audit-components.mjs. Do not edit manually."` in the JSON output.

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| Script path wrong from package.json | Use relative path `scripts/audit-components.mjs`; npm runs from package root |
| JSON import stale after re-audit | Vite HMR should pick up JSON changes in dev mode; production requires rebuild |
| CI environment doesn't have docs app | `audit:check` only depends on component package, not docs |

## Security Considerations

- No secrets or credentials
- Script only reads source files, writes one JSON file
- CI mode is read-only (reads JSON, checks results)
